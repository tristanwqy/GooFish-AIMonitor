GooFish-AIMonitor 桌面版 —— 双击安装,打开就是一个**独立的应用窗口**(自带图标和应用名,不是浏览器标签页)。Chromium 已打进包里,装完离线即用;数据存在你自己的用户目录,不外传。

## 本次更新(v0.1.3)

- **修复 AI 审核「测试通过却全跳过」**:`gemma`、`qwen3` 等**推理模型**会先输出大段思考(reasoning)再给 JSON,一次把整批商品塞过去会把 token 额度耗在思考上、正文为空 → 解析失败被静默放行(显示「审核未运行」)。现在:① 审核**分小批**(每批 5 条)调用,② 自动剥离 `<think>` 思考、正文为空时兜底,③ **日志**写到 `数据目录/app.log` 方便排查,④ 默认放宽超时/额度(60s / 4000 tokens)。
- **「测试 LLM」改为诚实判定**:只有真的拿到可解析裁决才显示「可用」;连上了但正文为空 / 不是 JSON 会直接告诉你原因和怎么修(如推理模型请把「最大 tokens」调到 8000)。测试还会用你真实的「AI 审核要求」来压,贴近实际。
- 若用本地/推理模型:设置里把 **最大 tokens 调到 8000、超时 ≥120s**,点「测试 LLM」显示「AI 审核可用」即可,再用「一键 AI 审核」补审旧的。

### v0.1.2

- **修复 macOS 包「已损坏,无法打开」**:打包后整包重新 ad-hoc 签名,下载到别的 Mac 不再被判损坏(降级为普通的「未验证开发者」拦截,右键→打开即可)。DMG 也加了「拖到 Applications」安装入口。
- **修复收藏夹空 / 一个推荐都没有**:打包后抓取读不到扫码登录态(登录态文件路径错锚到进程工作目录),导致每轮都按「未登录」退出。现已锚定到数据目录,登录后收藏与推荐正常。
- **设置页新增「测试 LLM」按钮** + **推荐页新增「一键 AI 审核」按钮**。

## 下载

- **macOS · Apple 芯片(M1/M2/M3/M4)**:下 `*-macos-arm64-*.dmg`(暂不支持 Intel 芯片 Mac)
- **Windows 10 / 11(64 位)**:下 `*-Setup-*.exe`

## 第一次打开(重要)

这是未签名的个人自用包,首次打开系统会拦一下,放行即可:

- **macOS**:打开 `.dmg` 后把图标**拖进「应用程序」**;再到「应用程序」里 **右键图标 →「打开」**,在弹窗里点「打开」(只需第一次)。或到「系统设置 → 隐私与安全性」点「仍要打开」。
  - 万一仍提示「已损坏」,在「终端」跑一行解掉隔离标记即可:
    `xattr -dr com.apple.quarantine "/Applications/GooFish-AIMonitor.app"`
- **Windows**:出现「Windows 已保护你的电脑」时,点「更多信息 → 仍要运行」。

打开后扫码登录即可使用。
